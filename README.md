# TA招聘系统 - 完整前端页面

## 项目概述
这是一个完整的TA招聘系统前端项目，包含10个主要页面，支持TA、MO、Admin三个角色的不同功能。整个系统采用响应式设计，提供了现代化的用户界面。

## 项目结构

```
PAGE/
├── index.html (项目入口 - 建议从login.html开始)
├── css/
│   └── style.css (全局样式 - 包含所有组件样式)
├── js/
│   └── script.js (全局JavaScript函数库)
├── assets/ (资源文件夹 - 预留)
│
├── 通用页面
├── login.html (系统登录页)
├── profile.html (个人中心页 - 所有角色通用)
│
├── TA专属页面
├── ta-profile.html (TA个人档案创建页 - 多步骤表单)
├── ta-positions.html (职位浏览页 - AI推荐+筛选)
├── ta-applications.html (我的申请中心)
│
├── MO专属页面
├── mo-positions.html (职位管理页)
├── mo-review.html (申请审核中心)
│
└── Admin专属页面
    ├── admin-analytics.html (数据统计与监控)
    ├── admin-users.html (用户管理 & 数据导出)
    └── admin-logs.html (系统日志与审计)
```

## 页面详细说明

### 1. 系统登录页 (login.html)
- **功能**: 用户身份认证
- **特点**:
  - 角色选择下拉框(TA/MO/Admin)
  - 账号和密码输入
  - 密码显示/隐藏切换
  - 表单验证提示
  - 错误3次后账号锁定5分钟
  - 忘记密码链接
  - 响应式居中卡片设计

### 2. 个人中心页 (profile.html)
- **功能**: 用户信息管理和通知
- **特点**:
  - 左侧用户信息卡片(头像、基本信息、档案状态)
  - 三个标签页:
    - 个人信息(编辑基本资料)
    - 密码修改(原密码、新密码确认)
    - 通知中心(已读/未读通知管理)
  - 响应式左右布局

### 3. TA档案创建页 (ta-profile.html)
- **功能**: TA多步骤档案创建
- **特点**:
  - 4个步骤: 基础信息 → 技能与时间 → 简历上传 → 确认提交
  - 步骤进度条(可视化步骤状态)
  - 步骤1: 年级、专业、GPA、邮箱
  - 步骤2: 技能标签多选、时间表格选择
  - 步骤3: PDF简历上传、拖拽支持、文件预览
  - 步骤4: 信息汇总确认
  - 支持前后移动和验证

### 4. 职位浏览页 (ta-positions.html)
- **功能**: 浏览和搜索职位
- **特点**:
  - 顶部AI推荐栏(横向列表、匹配度显示)
  - 左侧筛选面板(课程、专业、技能、状态、招聘人数)
  - 右侧职位列表(卡片式展示)
  - 每个职位显示:
    - 职位名称、课程、标签
    - 招聘人数、待遇、时间
    - 截止倒计时(黄色提醒)
    - 查看详情和立即申请按钮
  - 职位详情模态框

### 5. 我的申请中心 (ta-applications.html)
- **功能**: 管理TA的职位申请
- **特点**:
  - 按状态筛选(全部、待审核、已通过、已拒绝、已撤销)
  - 申请列表表格:
    - 职位名称、MO、申请时间、招聘人数
    - 状态标签(不同颜色区分)
    - 截止日期
  - 操作按钮(状态相关)
  - 设置申请优先级功能(志愿选择)
  - 支持批量操作

### 6. 职位管理页 (mo-positions.html)
- **功能**: MO发布和管理职位
- **特点**:
  - 发布新职位按钮
  - 职位状态筛选
  - 职位列表表格:
    - 职位ID、课程、招聘人数/已申请/已录用
    - 发布和截止时间
    - 状态标签
  - 每个职位的操作(编辑、关闭、查看申请、重开)
  - 发布/编辑职位的多步骤表单
  - 查看申请者列表(AI筛选支持)

### 7. 申请审核中心 (mo-review.html)
- **功能**: MO审核TA的申请
- **特点**:
  - 左侧职位选择列表
  - AI筛选栏(匹配度、申请优先级)
  - 右侧申请者列表(卡片式展示)
  - 每个申请者卡片显示:
    - QM ID、姓名、专业
    - 匹配度百分比
    - 匹配技能(重叠部分红色突出)
    - 申请优先级标签
  - 查看简历和档案按钮
  - 通过/拒绝审核操作
  - 支持批量操作

### 8. 数据统计页 (admin-analytics.html)
- **功能**: 系统数据监控和统计分析
- **特点**:
  - 4个统计卡片(总发布职位数、总申请数、总录用TA数、未招满职位数)
  - Chart.js图表:
    - 职位发布趋势(折线图)
    - MO工作量统计(柱状图)
    - 录用专业分布(饼图)
  - 两个数据表格:
    - 全职位列表
    - 录用TA列表
  - 标签页切换

### 9. 用户管理 & 数据导出页 (admin-users.html)
- **功能**: 用户账号管理和数据导出
- **特点**:
  - 标签页1: 用户管理
    - 筛选栏(角色、状态、搜索)
    - 用户列表表格:
      - QM ID、姓名、角色、创建时间
      - 最后登录时间、状态、操作
    - 停用/启用用户按钮
  - 标签页2: 数据导出
    - 导出配置卡片:
      - 数据类型选择(职位、申请、录用、用户、日志)
      - 时间范围选择
      - 条件筛选
    - 预览数据和导出按钮
    - 导出历史记录(带下载链接)
    - 数据预览表格

### 10. 系统日志与审计页 (admin-logs.html)
- **功能**: 系统操作日志管理
- **特点**:
  - 高级筛选栏:
    - 操作类型下拉
    - 操作人搜索
    - 时间范围选择
    - 操作结果筛选
  - 日志列表表格:
    - 日志ID、操作人信息(QM ID、姓名、角色)
    - 操作类型标签
    - 操作内容
    - 操作时间
    - 结果状态(成功/失败)
  - 导出操作日志
  - 分页功能
  - 查看详情模态框

## Git 自动暂存与同步到 GitHub 分支

如果你希望“每次修改后自动保存到 Git（自动暂存）”，并且随时可以“一键提交并同步到 GitHub 分支”，可以使用本项目新增的两个脚本。

### 1. 首次初始化（只做一次）
1. 在项目根目录初始化仓库（如果还没初始化）：
  - `git init`
2. 绑定远程仓库：
  - `git remote add origin <你的GitHub仓库地址>`
3. 切换到目标分支（例如 `dev`）：
  - `git checkout -b dev`
4. 首次推送并建立上游：
  - `git push -u origin dev`

### 2. 启动自动暂存监听
在项目根目录运行：

`powershell -ExecutionPolicy Bypass -File .\git-autostage-watch.ps1`

说明：
- 脚本会监听文件变更并自动执行 `git add -A`
- 使用了消抖机制，默认 3 秒内连续修改只会触发一次暂存
- 按 `Ctrl + C` 可停止监听

### 3. 一键提交并推送当前分支
在项目根目录运行：

- 带提交信息：`git-commit-push.bat "feat: 更新登录逻辑"`
- 不带提交信息：`git-commit-push.bat`（会自动生成时间戳消息）

脚本行为：
- 自动 `git add -A`
- 若有变更则执行 `git commit`
- 自动推送到 `origin/<当前分支>`

### 4. 推荐工作流
1. 开一个终端常驻运行 `git-autostage-watch.ps1`
2. 正常修改代码
3. 需要保存里程碑时执行 `git-commit-push.bat "你的提交信息"`

这样可以做到“修改后自动进 Git 暂存区 + 随时一键提交并同步到 GitHub 分支”。
